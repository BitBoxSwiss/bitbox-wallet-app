We use Weblate as a translation service.

Language files are stored in `frontends/web/src/locales`.

The reference language is `en`.

## Workflow

Weblate pulls from the `master` branch of
[BitBoxSwiss/bitbox-wallet-app](https://github.com/BitBoxSwiss/bitbox-wallet-app/tree/master) and
pushes translated commits to
[BitBoxSwiss/bitbox-wallet-app-weblate](https://github.com/BitBoxSwiss/bitbox-wallet-app-weblate/tree/master).

Repo maintainers manually merge from the weblate repo into the main repo.

## For translators

Translators are given access to the Weblate instance directly.

## For developers

Keys can be added to or removed from `frontends/web/src/locales/en/app.json`. Keys for other
languages **should not** be added or removed manually as that could cause merge conflicts in weblate.

Existing keys **should not** be modified directly, but sometimes it is still useful to modify keys
anyway. Reasons can be for review/testing purposes or to keep track which keys have to be updated
after the PR was merged. However try to avoid this to reduce the chance of merge conflicts.

Ideally you only add, edit or remove keys / translations for EN in `/en/app.json` and edits for all
other languages are handled in Weblate.
